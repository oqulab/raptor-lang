# How to Contribute

## Reporting Bugs

- Use GitHub Issues
- Include: Raptor version, code snippet that reproduces the bug, expected vs actual behavior
- Search existing issues first

## Proposing Features

- Open a Discussion first to align on the feature
- PRs without prior discussion may be closed

## Pull Requests

1. Fork the repo
2. Create a branch: `feature/your-feature` or `fix/your-bug`
3. Write tests for new functionality
4. Ensure `./gradlew :raptor:test` passes
5. Open PR against `main`
6. Keep PRs focused — one change per PR

## Code Style

- Follow Kotlin conventions
- Use KDoc for public API
- Keep functions small and focused
- Add tests for interpreter behavior changes
